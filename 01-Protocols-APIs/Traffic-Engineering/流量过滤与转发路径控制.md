## 流量过滤与转发路径控制

传统的路由转发原理是首先根据报文的目的地址查找路由表，然后进行报文转发。随着业务的发展，用户更加希望能够在传统路由转发的基础上根据自己定义的策略进行报文转发和选路

为提高网络安全性，用户希望能够控制进入网络的报文，将没有权限进入网络或存在安全隐患的报文隔离在网络边界

### PRB Policy-Based Routing

PRB与Route-Policy类似，由多个节点组成，每个节点由条件语句与执行语句组成

```
policy-based-route PBR permit node 10
 if-match acl 2000
 apply next-hop 10.0.0.1
```

permit表示对满足匹配的报文进行策略路由

deny表示对满足匹配的报文不进行策略路由

*Route-Policy是对路由信息进行过滤用的，PRB是对报文直接进行操作*

### MQC Modular QoS Command-Line Interface

模块化QoS命令行

```
流策略
  流分类
  流行为

  流分类
  流行为
```

*流策略不同于PBR，PBR只能调用在三层接口，而流策略支持调用在二层接口*

配置方式如下：

```
traffic classifier aaa

traffic behavior aaa

traffic policy aaa # 华三用qos policy
  classifier aaa behavior aaa
```

### 流量过滤

在实际网络中常常需要丢弃不信任报文，或者要求不同部门间不能互相访问，为此产生了流量过滤工具

**Traffic-Filter**

可以在边界路由器的出入接口（**只能在接口上部署**）上部署，用来阻断访问

华三应该是packet-filter，华为是traffic-filter

**MQC**

也可以使用MQC阻断访问

例如：在acl中permit对应访问信息，然后在流分类中if-match此acl，再在liu'xing'wei'zhong

### 重点

全局部署的路由策略用于过滤本地始发的流量，接口上部署的路由策略用于接口入方向的流量
