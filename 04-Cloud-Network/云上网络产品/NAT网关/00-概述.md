NAT 网关（NAT Gateway）是阿里云全托管的网络地址转换网关，通过转换和隐藏云服务地址，防止地址直接暴露，提升网络安全性。

## 产品类型

NAT 网关分为公网 NAT 和 VPC NAT 网关，公网 NAT 网关支持绑定弹性公网IP，实现公网访问。VPC NAT 网关用于 VPC 内的地址转换。

### **公网 NAT 网关 - 公网访问**

公网 NAT 网关可以将 VPC 内的私网 IPv4 地址转换为 EIP，实现多服务器共享EIP访问公网。

| **服务器访问公网** 使用公网 NAT 网关的 SNAT 功能，可实现多个 ECS 实例共享 EIP 访问公网，节省成本的同时提升安全性。 ![image](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/6381714871/CAEQYhiBgMDQlPbHyhkiIDUwNGIzODM3NTBiNzRjYzQ4ZjYyMjcxOTFmMGQ5NTUz4811797_20250618105744.796.svg) | **多 VPC 共用 NAT 访问公网** 使用 VPC 对等连接或云企业网 CEN 实现 VPC 互连，可实现多个 VPC 共用公网 NAT 网关访问公网。 ![image](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/6381714871/CAEQYhiBgICOgPrHyhkiIGNhNzE0ZTc5NTNiZjQxMWE5NWZhNGEwZTliMzUwZDEy5358967_20250624172026.483.svg) |

#### **公网 NAT 网关与ECS 直绑 EIP 的区别**

与直接为 ECS 绑定 EIP 相比，公网 NAT 网关提供了共享 EIP、统一出口管理等能力。

| **维度** | **ECS 直绑 EIP** | **公网 NAT 网关** |
| --- | --- | --- |
| **EIP 用量** | 每个 ECS 绑定1个 EIP，EIP 数量随 ECS 实例数量增长 | 多个 ECS 共享少量 EIP，支持绑定多个 EIP 叠加带宽 |
| **出口安全** | 每个 ECS 对外使用独立 EIP，源 IP 可见 | 出口 IP 收敛为少量地址，隐藏后端实例的源 IP |
| **运维能力** | EIP 分散管理，无法统一监控和限速 | 网关集中管理，支持流量监控、带宽限速和策略配置 |
| **适用场景** | 少量 ECS 或容器，无出口管控要求 | 多个 ECS 或容器，需统一出口或流量管控 |

### **VPC NAT 网关 - 私网访问**

VPC NAT 网关可以将 VPC 内的私网 IPv4 地址转换为 NAT IP，解决地址冲突网络的互访或满足使用指定地址访问的诉求。

| **私网冲突解决** 网段重叠的 VPC 无法互连，可以使用 VPC NAT 网关对冲突的私网地址进行转换。 ![image](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/6381714871/CAEQYhiBgMCJo_jHyhkiIDc5NGI3MmJjOGEzOTQ5Yzg4MmM1M2MxMjIxZWJiZTI54585827_20240820105942.020.svg) |
| --- |
| **指定地址访问** 金融证券等受监管行业，云上业务使用 VPC NAT 网关，确保通过固定的、指定的私网IP地址访问本地IDC。 ![image](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/6381714871/CAEQYhiBgMCzi57gyRkiIDIyNzIxY2I1YmJkZDQ4MDY5ZTExYjkzM2RkZWNhY2Ni5286340_20250618134928.042.svg) |

## 性能与高可用

### **高可用能力**

NAT 网关支持跨可用区容灾和单可用区容灾两种部署模式，创建时通过**容灾类型**选择。

> 单可用区容灾模式已在 NAT 网关支持的所有地域开放，如需使用请联系客户经理申请。

| **跨可用区容灾** NAT 网关跨可用区冗余部署，单可用区故障时自动容灾切换，适用于 ECS 等实例分布在多个可用区、需要共享同一 NAT 网关访问公网。 ![image](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/7381714871/CAEQbxiBgIDW8pmb8BkiIDE1YzZmMDI3NDU3MzQ4MWFhN2EwYmI1ZTYzMjNiMTQ44811797_20250618105744.796.svg) | **单可用区容灾** NAT网关部署在单一可用区内，只保障本可用区高可用，适用于业务集中在某一个可用区、使用独立 NAT 网关访问公网。 ![image](data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAQAAAC1HAwCAAAAC0lEQVR42mNkYAAAAAYAAjCB0C8AAAAASUVORK5CYII=) |
| --- | --- |

|     | **跨可用区容灾（默认）** | **单可用区容灾** |
| --- | --- | --- |
| **部署方式** | 主备两个可用区各部署一套实例，备可用区由阿里云选择 | 仅在用户指定的可用区内部署，可用区内设备级冗余 |
| **容灾能力** | 可用区故障时自动切换 | 阿里云确保本可用区内的高可用能力 |
| **费用** | 基准价 | 实例费约为跨可用区的 50%，CU 费约为 80% |
| **适用场景** | ECS 等实例分布在多个可用区、需要共享同一 NAT 网关访问公网 | 业务集中在某一个可用区、使用独立 NAT 网关访问公网 |

### **自动弹性**

NAT 网关提供跨可用区容灾和单可用区容灾两种部署模式，性能指标如下。

| **部署模式** | **性能指标** | **新建连接速率（CPS）** | **吞吐量（包括入流量和出流量）** | **并发连接数** | **包处理速率（PPS）** |
| --- | --- | --- | --- | --- | --- |
| 跨可用区容灾 | 初始指标 | 2万  | 5 Gbps | 50万 | 80万 |
| 弹性上限 | 10万 | 15 Gbps | 200万 | 250万 |
| 单可用区容灾 | 初始指标 | 2万  | 10 Gbps | 50万 | 80万 |
| 弹性上限 | 10万 | 20 Gbps | 200万 | 250万 |

为保证性能和稳定性，NAT网关采用分布式高可靠集群架构，所有流量将平均分配到集群中的多台系统服务器上进行转发。因此上述吞吐量和包处理速率仅在多会话情况下才能达到最大弹性上限，单会话的吞吐量和包处理速率受单台系统服务器性能限制。建议您先进行业务压测，以了解实例的性能表现，并配置相应监控。

> 单会话的吞吐量上限=吞吐量弹性上限÷集群中服务器的个数

> 单会话的PPS上限=包处理速率弹性上限÷集群中服务器的个数

当NAT网关的性能超过最大指标时，业务访问将面临丢包风险。如需更高性能，请联系商务经理进行申请。

在实际业务场景中，NAT网关的性能受到实例包长（网络数据包的平均大小）、连接类型（长连接或短连接）、网络架构等多种因素的影响。因此，实例的实际性能表现可能存在偏差。建议您结合自身业务特点，提前进行压力测试以评估实例性能，并合理配置监控项，以确保业务的平稳运行。
