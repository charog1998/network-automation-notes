## 1、安装和配置 Ansible
按照下方所述，在控制节点 control 上安装和配置 Ansible：
- [ ] 安装所需的软件包
- [ ] 创建名为 /home/greg/ansible/inventory 的静态清单⽂件，以满⾜以下要求：
- [ ] node1 是 dev 主机组的成员
- [ ] node2 是 test 主机组的成员
- [ ] node3 和 node4 是 prod 主机组的成员
- [ ] node5 是 balancers 主机组的成员
- [ ] prod 组是 webservers 主机组的成员
- [ ] 创建名为 /home/greg/ansible/ansible.cfg 的配置⽂件，以满⾜以下要求：
- [ ] 主机清单⽂件为 /home/greg/ansible/inventory
- [ ] playbook 中使⽤的⻆⾊的位置包括 /home/greg/ansible/roles
- [ ] ⾃定义的 collection ⽬录在 /home/greg/ansible/mycollection

### 实操过程

#### 切换到主控主机
- `ssh greg@control`

#### 安装ansible
- `sudo yum -y install ansible-core ansible-navigator`

#### 创建文件夹
- `mkdir -p /home/greg/ansible/roles`
- `cd /home/greg/ansible`
- `mkdir mycollection`

#### 创建配置文件
- `cat /etc/ansible/ansible.cfg`
- 从ansible这个配置文件里复制粘贴出下面这行
- `ansible-config init --disabled > ansible.cfg`
- *注：因为我们此时就在`/home/greg/ansible`目录下，所以可以直接写相对路径*

#### 修改配置文件
- `vim ansible.cfg`
- 在vim里用斜杠做查找，例如 `/inventory=`
- 查找到之后用 `yy` 复制一整行，按 `p` 粘贴然后再修改
- 粘贴出来之后只需要简单调整即可，具体修改内容如下：

```
[defaults]
inventory=/home/greg/ansible/inventory
remote_user=greg # 在受管主机上登陆的用户名
host_key_checking=False # 主机密钥检查
roles_path=/home/greg/ansible/roles:/usr/share/ansible/roles
collections_path=./mycollection/:.ansible/collections:/usr/share/ansible/collections
[privilege_escalation]
become=True # 表示需要提权为root用户
```

#### 确认已生效的配置文件
- `ansible --version`
- `ansible-galaxy list`
- 列出所有已安装的 galaxy

#### 创建主机组文件
- `vim inventory`

```yml
[dev]
node1
[test]
node2
[prod]
node3
node4
[balancers]
node5
[webservers:children]
prod
```

#### 确认主机组是否正确
- `ansible-inventory --graph`
- `ansible all -m ping -o`
- 加上 `-o` 可以让回显结果变成一行，加不加都行

#### 验证podman登录，镜像操作等
- `podman login utility.lab.example.com -u admin -p redhat`
- 镜像服务器域名，账户名密码在一开始的基本信息里有
- `ansible-navigator images`
- 用这行会下载一个镜像
- `ansible-navigator collections`
- 可以查看支持的模块
