## 4、使用 RHEL 系统角色
使⽤ RHEL 系统⻆⾊
安装 RHEL 系统⻆⾊软件包，并创建符合以下条件的 playbook /home/greg/ansible/selinux.yml ：
- [ ] 在 所有受管节点 上运⾏
- [ ] 使⽤ selinux ⻆⾊
- [ ] 配置该⻆⾊，配置被管理节点的 selinux 为 enforcing

### 实操过程
#### 安装系统角色
- `yum search role`
- 结果中找到`=============Name MAtched: role=============`这行，下面就写着包名,可以复制粘贴
- `sudo yum -y install rhel-system-roles.noarch`
- `ansible-galaxy list`
- 检查下各个角色目录下的`roles`，可以看到`/use/share/ansible/roles`这个目录下添加了很多系统角色
- *切到`/use/share/doc/rhel-system-roles/*`这些目录下能看到这些角色的使用文档*

#### 编写剧本
- `cp /usr/share/doc/rhel-system-roles/selinux/example-selinux-playbook.yml selinux.yml`
- 直接复制官方的例子过来
- 在`vars`里只保留前两项：`selinux_policy`、`selinux_state`
- 在任务里面只保留`execute the role and catch errors`
- *在vim里可以直接`:11,39d`，删除11到39行*
```yml
---
- hosts: all
  become: true
  become_method: sudo
  become_user: root
  vars:
    # Use "targeted" SELinux policy type
    selinux_policy: targeted
    # Set "enforcing" mode
    selinux_state: enforcing

  # Prepare the prerequisites required for this playbook
  tasks:
    - name: execute the role and catch errors
      block:
        - name: Include selinux role
          include_role:
            name: rhel-system-roles.selinux
      rescue:
        # Fail if failed for a different reason than selinux_reboot_required.
        - name: handle errors
          fail:
            msg: "role failed"
          when: not selinux_reboot_required

        - name: restart managed host
          reboot:

        - name: wait for managed host to come back
          wait_for_connection:
            delay: 10
            timeout: 300

        - name: reapply the role
          include_role:
            name: rhel-system-roles.selinux
```
#### 运行
- `ansible-playbook selinux.yml`
- 由于是使用RPM包装的角色，依然要用`playbook`运行剧本

#### 验证
- `grep ^SELINUX= /etc/selinux/config; getenforce`
- 先在本机上利用补全把这行敲出来运行一下
- 然后左右添加单引号，发送给所有主机验证下
- `ansible all -m shell -a 'grep ^SELINUX= /etc/selinux/config; getenforce'`
- 结果能看到配置文件中被修改为`SELINUX=enforcing`了，当前状态也是`Enforcing`
