### 实操过程

#### 查找文档
- `ansible-doc -l | grep yum`
  - 回显中会显示包含`yum`的模块，在想不起来模块名时很好用
- `ansible-doc yum_repository`
  - 可以查看`yum_repository`这个库的使用文档
  - 和VIM一样，用`/EXAMPLE+回车`可以查找，用`Nn`可以查找上下项
  - 可以开两个终端等下复制粘贴过来

#### 调vim的参数
- `echo set nu ts=2 et cuc sw=2 autoindent > ~/.vimrc`
- 这个敲完之后写`yaml`文档会特别舒服
> 小爱同学的解释：这条命令的作用是创建或覆盖用户主目录下（通常是/home/用户名）的.vimrc 文件，并向其中写入特定的配置内容。其中，"set nu"表示显示行号；"ts=2"表示设置制表符宽度为 2 个空格；"et"表示启用自动缩进；"cuc"表示连续行的自动缩进；"sw=2"表示设置换行时的缩进宽度为 2 个空格；"autoindent"表示启用自动缩进功能。这样就可以对 Vim 编辑器的一些行为进行配置。


#### 编写剧本
- `vim yum_repo.yml`
```yml
- name: repository
  hosts: all # 还有下面的tasks都是复数
  tasks:
    - yum_repository:
        file: EX294_BASE # 这里一定要记得按两下Tab，按一次只推两个空格
        name: EX294_BASE
        description: "EX294 base software"
        baseurl: http://content/rhel9.0/x86_64/dvd/BaseOS
        gpgcheck: yes
        gpgkey: http://content/rhel9.0/x86_64/dvd/RPM-GPG-KEY-redhat-release
        enabled: yes # 注意有个d，不是enable原型，是被动形式
    - yum_repository:
        file: EX294_STREAM
        name: EX294_STREAM
        description: "EX294 stream software"
        baseurl: http://content/rhel9.0/x86_64/dvd/AppStream
        gpgcheck: yes
        gpgkey: http://content/rhel9.0/x86_64/dvd/RPM-GPG-KEY-redhat-release
        enabled: yes
```
- 第二段可以在`vim`里用`8yy`复制8行再按`p`粘贴，省好多时间
- 小写`v`（可以选列）、大写`V`（可以直接选整行）然后上下移动光标选中多行，按`shift+，。`（也就是大于小于号）连按两次可以直接多行一起调缩进，万一缩进有问题用这个会特别方便。
- 这题里面的参数基本上就是按照题目里面的要求一行一行写就行。

#### 运行
- `ansible-navigator run yum_repo.yml -m stdout`
- 相当于以前的`ansible-playbook`
