## 12、创建 Web 内容目录
创建 Web 内容目录
按照下⽅所述，创建⼀个名为 /home/greg/ansible/webcontent.yml 的 playbook ：
- [ ] 该 playbook 在 dev 主机组中的受管节点上运⾏
- [ ] 创建符合下列要求的⽬录 /webdev ：
  - [ ] 所有者为 webdev 组
  - [ ] 具有常规权限： owner=read+write+execute， group=read+write+execute， other=read+execute
  - [ ] 具有特殊权限 ：设置组 ID
- [ ] ⽤符号链接将 /var/www/html/webdev 链接到 /webdev
- [ ] 创建⽂件 /webdev/index.html ，其中包含如下所示的单⾏⽂件： Development
- [ ] 在 dev 主机组中主机上浏览此⽬录（例如 http://172.25.250.9/webdev/ ）将⽣成以下输出：
    Development

### 实操过程

#### 编写
- `vim webcontent.yml`
```yml
---
- name: webcontent
  hosts: dev
  roles:
  - apache # 用前面创建过的apache角色来开启http服务
  tasks:
    - name: Create a directory if it does not exist
      ansible.builtin.file:
        path: /webdev
        group: webdev
        state: directory
        mode: '2775' # 设置组id，是2
    - name: Create a symbolic link
      ansible.builtin.file:
        dest: /var/www/html/webdev
        src: /webdev
        state: link
    - name: Copy using inline content
      ansible.builtin.copy:
        content: 'Development'
        dest: /webdev/index.html
        setype: httpd_sys_content_t # 用这个选项指明SELinux的文件类型为apache静态文件
```
- `dev`这个主机组目前还没有打开`http`服务，记得要用`roles`打开
- 记得把`index.html`设置成`httpd_sys_content_t`
- 如果不设置的话，`curl http://node1/webdev/`会报403错误：
```html
<!DOCTYPE HTML PUBLIC "-//IETF//DTD HTML 2.0//EN">
<html>
<head>
    <title>403 Forbidden</title>
</head>
<body>
    <h1>Forbidden</h1>
    <p>You don't have permission to access this resource.</p>
</body>
</html>
```
- `template`模块也有这个选项，但是那个不用设置居然也可以

#### 运行
- `ansible-navigator run webcontent.yml -m stdout`

#### 验证
- `curl http://node1/webdev/`或者浏览器直接打开
- 能看到 Development

#### 备注
##### 设置用户id
- 用`chmod 4777`或者`chmod u+s`
- 只能设置在二进制程序文件上，设置后普通用户在执行这个文件时也能获得和文件所有者一样的权限，通常用在一些需要root权限的程序上，例如`/usr/bin/passwd`
##### 设置组id
- 用`chmod 2777`或者`chmod g+s`
- 设置之后的效果就是在这个目录下创建的子目录的属组都是这个父目录的属组，跟创建者无关
##### 粘滞位（Sticky Bit）
- 用`chmod 1777`或者`chmod o+t`
- 设置之后这个目录下大家都可以写文件，但是只能删除自己创建的文件而无法删除其他用户创建的文件
- 正常来讲有写权限就应该能删除，但是这个粘滞位起到了一个防删除的作用
- 例如`/tmp`
