## 11、修改文件内容
修改文件内容
按照下⽅所述，创建⼀个名为 /home/greg/ansible/issue.yml 的 playbook ：
- [ ] 该 playbook 将在 所有清单主机 上运⾏
- [ ] 该 playbook 会将 /etc/issue 的内容替换为下⽅所示的⼀⾏⽂本：
  - [ ] 在 dev 主机组中的主机上，这⾏⽂本显示 为： Development
  - [ ] 在 test主机组中的主机上，这⾏⽂本显示 为： Test
  - [ ] 在 prod 主机组中的主机上，这⾏⽂本显示：Production

### 实操过程

#### 编写
- `vim issue.yml`
```yml
---
- name: modify file 1
  hosts: dev
  tasks:
    - name: Content 1
      ansible.builtin.copy:
        content: 'Development'
        dest: /etc/issue
- name: modify file 2
  hosts: test
  tasks:
    - name: Content 2
      ansible.builtin.copy:
        content: 'Test'
        dest: /etc/issue
- name: modify file 3
  hosts: prod
  tasks:
    - name: Content 3
      ansible.builtin.copy:
        content: 'Production'
        dest: /etc/issue
```
- **上面这个写法在测试环境判分不通过，换成下面的写法好使了，不知道啥原因**
```yml
---
- name: issue
  hosts: all
  tasks:
    - name: Content 1
      ansible.builtin.copy:
        content: 'Development'
        dest: /etc/issue
      when: inventory_hostname in groups.dev
    - name: modify file 2
      ansible.builtin.copy:
        content: 'Test'
        dest: /etc/issue
      when: inventory_hostname in groups.test
    - name: modify file 3
      ansible.builtin.copy:
        content: 'Production'
        dest: /etc/issue
      when: inventory_hostname in groups.prod
```

#### 运行
- `ansible-navigator run issue.yml -m stdout`

#### 验证
- `ansible all -a 'cat /etc/issue'`
- 结合`ansible-inventory --graph`确认每个节点的内容是否正确
